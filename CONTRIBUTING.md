# Contributing Guidelines

Thank you for taking the time to improve this companion resource hub!

## What to Contribute

- **Bug fixes**: Correcting incorrect code snippets, register names, or configuration values
- **Documentation**: Improving clarity of existing notes and reference sheets
- **New content**: Adding peripheral reference docs for topics not yet covered
- **Wiring diagrams**: Adding circuit diagrams or pinout tables

## How to Contribute

1. **Fork** this repository
2. **Create a branch**: `git checkout -b fix/uart-typo` or `docs/add-i2c-reference`
3. **Make your changes** — follow the style of existing `.md` files
4. **Commit** with a clear message: `fix: correct UART baud rate formula`
5. **Open a Pull Request** with a short description of what you changed

## Style Guidelines

- All content must be in **English**
- Use **code blocks** with language tags (` ```c `) for all code samples
- Keep code samples **concise** and focused on one concept
- Reference the **official ST HAL documentation** when in doubt
- For register-level code, cite the relevant section of the **RM0368** reference manual

## Reporting Issues

Found an error? [Open an issue](https://github.com/duongvanhoawork-lang/STM32_NUCLEO_F401RE/issues) with:
- The file and line number
- The problem
- The correct value/information (with source if possible)

## Code of Conduct

- Be respectful and constructive
- Contributions must be original or properly attributed
- Do **not** include any private student data or graded submission content
